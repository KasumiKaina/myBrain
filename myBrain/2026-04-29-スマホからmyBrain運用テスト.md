# スマホからmyBrain運用テスト

> created: 2026-04-29
> tags: #気づき #運用 #obsidian

## 話題

スマホブラウザから claude.ai/code 開いて、myBrain Vault がどこまで素直に運用できるかのテスト。partnerBrainの読み込み・ノート生成・push までワンセットでいけるか確認したかった。

## 結論・方針

ひとまず動作OK。
- `partnerBrain/summary/` も `details/` も Read で普通に取れる
- `obsidian.skill` ZIP も `unzip -p` で中身参照できる(Skill auto-loadは効かんけど代替で十分)
- 作業ブランチ `claude/test-obsidian-mobile-partner-xxx` が harness で切られてるから main直push事故も起きにくい

## 蹴った選択肢

- **モバイルでObsidian Mobile直編集**: 出先メモには使えるけど、Claudeに会話文脈渡しながらノート化するフローは claude.ai/code 経由の方が圧倒的に楽。両立運用でいく。
- **Web版Claudeで `/mnt/user-data/outputs/` 経由**: 手動でVaultに配置する手間が出る。Claude Code (Web) なら git までやってくれるので不採用。

## 判断軸

- 「思考ログ→ノート→push」までの摩擦をどれだけ減らせるか
- 出先(電車・移動中・飯屋待ち)でもサッと起動できるか
- PC帰宅後に `git pull` 一発で同期できる単純さ

## モヤモヤ・揺れ

- Obsidian Mobile + obsidian-git で見るには main マージが必要、ってのが一手間。作業ブランチのまま放置するとモバイル側に降ってこない。
- スマホでがっつり長文打つのはやっぱ厳しい。音声入力+Claude整形のフローも試してみたい。
- スキル auto-load 効かんのは想定通りやけど、毎回 `unzip -p` 走らせるのちょい冗長。

## 気づき・ツッコミ

- 「第二の自分」を育てるための入力パイプラインが、スマホからでも回る状態になったの地味にデカい。仕事の昼休みとかドライブ休憩中の思考をそのまま流し込める。
- harness が勝手にブランチ切ってくれる仕組み、main直push禁止ルールと相性ええな。事故防止が運用ルールじゃなく仕組み側に落ちてる。
- partnerBrainが取れる=人格コンテキスト持ったまま会話できる、ってことやから、ノートの口調ブレも減る。これは本来の狙い通り。
