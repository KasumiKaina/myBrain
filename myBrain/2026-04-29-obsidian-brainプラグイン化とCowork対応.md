# obsidian-brainプラグイン化とCowork対応

> created: 2026-04-29
> tags: #判断 #モヤモヤ #気づき #Cowork #Obsidian

## 話題

前に Claude Code 向けに作った obsidian スキル、Cowork でも動くようにしたかった。
ただコピペで済むのか、それとも何か変換いるのか、そもそも Cowork でスキル入れるのどうやるのか、全部分かってなかったから一個ずつ潰した。

## 結論・方針

`obsidian-brain` っていうプラグインに包んで両環境対応にした。

- **Claude Code**: そのまま自動 push（Git Credential Manager 任せ）
- **Cowork**: ノート配置までやって、PowerShell コマンドブロックを毎回表示。それをコピペして手動 push

スラコマ `/obsidian` も残した。Claude Code 側で使える。Cowork は自然言語（「ノート化して」とか）でトリガー。

## 蹴った選択肢

- **Cowork から自動 push（PAT 埋め込み方式）**
  → 一回は試した。PAT 発行 → remote URL に埋め込み → 認証は通った。けど git の lock ファイルが Cowork サンドボックスから消せない問題で詰む。FUSE マウントの permission 周りの相性悪い。あきらめた。
- **PAT を別ファイルに置いて読み込む方式**
  → 一段面倒。個人運用ならそこまで凝らなくていい。
- **`/obsidian` を Cowork でも実装**
  → Cowork はそもそも自然言語前提だから、無理に作っても発火しにくい。
- **.gitignore を UTF-16 のまま放置**
  → git に読まれてなかった。UTF-8 に直した。

## 判断軸

- **確実性 > 自動化**。不安定な自動 push より、確実に動くコピペの方がストレス少ない。
- **両環境で使える形にする**。Claude Code 側を犠牲にしたくない。
- **MVP 思考**。重複検出とか MOC 自動生成は後回し。
- **個人マシン＆プライベートリポなら PAT 埋め込みも許容**って判断したけど、結局使わない方式に落ち着いた。

## モヤモヤ・揺れ

- Cowork から push まで一気通貫したかった気持ちは残ってる。PowerShell 毎回開くの、頻度上がるとダルくなる気がする。
- マウントの permission 問題、根本解決できてない。今後似たケースで同じところで詰まりそう。
- `.git/index.lock` の問題、今回限りなのか毎回出るのか分からない。何度か試して傾向見たい。
- PAT 結局埋め込んだけど使わない方式になった。設定だけ残ってるの気持ち悪いっちゃ気持ち悪い（plain URL に戻すかは保留）。

## 気づき・ツッコミ

- Cowork のマウント、思ってたより制約あった。lock ファイル消せないのは想定外。
- `present_files` ツールが scratchpad の自動コピー効かなかった謎。結局 computer:// リンクで解決したけど、なんで？
- Claude（Cowork側）の Write tool で UTF-8 のつもりで書いたら UTF-16 になってたの謎い。bash の printf で書き直したら正常。何か変換挟まってる？
- 最初「フォルダ接続から」って言ったけど、結果的にプラグイン化のフェーズが一番時間かかった。読みが甘い。
- スキルが「Coworkでも基本いけるよ！」って即答してきたけど、その後出てきた制約のリスト見ると「基本」のとこで盛りすぎだった気もする。
- Claude Code 用に作ったスキルを「ほぼそのまま使える」と思ってたけど、実際はパス分岐とか push 方式とか結構手を入れた。雛形があったから速かったのは事実。
