# Cowork環境でobsidianスキル動作テスト

> created: 2026-04-29
> tags: #気づき #環境構築 #obsidian

## 話題

obsidianスキルが Cowork 環境でちゃんと動くかテストしたかった。
Claude Code では動いてたけど、Cowork は別物だから挙動が違う可能性あった。

## 結論・方針

動いた。Vaultフォルダ(`C:\Users\Jasminiums_\Documents\obsidian\myBrain`)を接続すれば、ノート配置までは普通にいける。
push は SKILL.md にある通り PowerShell で手動実行する運用。

ついでにマシンごとにパスが違う問題も解決:
- SKILL.md に個人PC(`Jasminiums_`)/会社PC(`TORIGOE`)の環境表を追加
- 本文中のパスは `<VAULT_ROOT>` プレースホルダで統一
- 実行時にユーザー名から判別する仕様にした

これで会社PCでも個人PCでも同じスキルでいける。

## 蹴った選択肢

- outputs にダミーノート吐いてフォーマット確認だけする
  → せっかくなら本番ルートで動作確認したい派だった
- ArsMagia フォルダ側に置く
  → そもそも間違いマウントだったので却下

## 判断軸

- 本番フローで動くか確認したかった(MVPの動作確認)
- ダミーじゃなくテスト記録自体をノートにすれば1石2鳥
- パス問題の解決方針は「環境別マッピング表」を採用。partnerBrain依存(疎結合崩れる)とプレースホルダ毎回確認(面倒)を蹴って、SKILL.md単独で完結する形に

## モヤモヤ・揺れ

- ~~SKILL.md のパスが `C:\Users\TORIGOE\...` になってる問題~~ → 解決済み(環境別マッピング表で対応)
- Cowork 側からの git push が詰まる問題、PowerShell 手動運用で回避してるけど、ちょっと面倒なのは残る。
- 新たな環境(別PC/別ユーザー名)が出てきたら都度 SKILL.md 更新が必要。当面は2台運用なので問題ないけど、増えたら別の方式(環境変数 or partnerBrain逃がし)検討するかも。
