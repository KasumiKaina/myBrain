# 思考スタイル

## 心がけること

- ツッコミを入れる：違和感あればストレートに言う
- 否定意見を必ず検討：肯定だけで終わらせない
- 構造化して返す：箇条書き・段階分け・比較を活用
- 仮説と確度を分ける：「これは確実」「これは推測」を明示

## Kainaの思考特性

- 抽象→具体への落とし込みが早い
- 複数案の比較検討を好む
- 完璧を目指すより、走りながら改善する派
- メンテコスト・運用継続性を重視
- 空間記憶が強い（一度行った場所はだいたい覚える）
- 道路構造・地形・建造物への観察癖（Factorioの工場設計好きと地続き）
- **何事もつい考える癖がある**（分析・考察モードがデフォルトで走る）
  - 漫画・アニメを見ても感情考察が走る → 能動消費になり疲れる
  - 整体は「翌日痛みが残るかで技術評価」、譜面は「構造美で語る」、道路は「設計と観察」、すべて同じ系列
  - 「ただ消費する」モードに切り替えにくい → だからFactorio(脳のストレッチ)が貴重

## コミュニケーション・記録スタイル

- 能動的文章化（日記・レポート）は苦手、続かない
- 受動的自動記録（GPS走行ログ等）は数年単位で続く
- 仕事の伝達も口頭派 > レポート派
- ただし**対話形式**なら自然に思考が出せる
- → Claude側は「対話で引き出して文書化する」流れが刺さる（obsidianスキルの運用思想とも一致）
- **読書も苦手**（ほとんど本を読んだことがない、小学校の読書タイムも嫌い）
  - 文字情報の能動的摂取が苦手
  - 情報収集は対話・観察・体験ベースが中心
  - → Claude側は「本読んで」「ドキュメント当たって」系の提案は刺さりにくい。要点を口頭で渡すか対話で展開する方がよい
- **漫画・アニメも最近は見ない**（昔は見ていたが、見ること自体に疲れを感じるようになった）
  - 物語コンテンツの能動的消費に対する許容認知負荷が下がっている
  - Factorio(何も考えずできる)やSDVX(リズム・パターン)系の「ストレッチ・反復型」コンテンツの方が今は刺さる傾向

## 注意・フィルタ機構

- **カクテルパーティー効果が標準より弱い**(聴覚的選択的注意の弱さ)
  - 居酒屋で別テーブルの話題聞こえる、オフィスで他人の電話に意識持っていかれる、突発音に苛立つ
- 「思考癖がデフォで走る」と地続き = **入ってきた情報を全部処理する癖**が音にも適用されてる(道路観察癖と同系列)
- 言語処理リソースが特に占有されやすい(自分の会話/電話中に他人の言語信号が入ると集中切れる)
- → 集中作業の場所/時間帯提案、会議・思考タスクの構成提案時に**音環境**を考慮する
- 詳細: [[../details/attention_profile]]

## 制約・自由度の好み

- **自由度が高すぎると逆に困る**タイプ
  - 「なんでもいい」が一番困る
  - TRPGのキャラメイクも事前ハンドアウトが少ない方が苦手
- **制約があった方が動ける・アウトプットが出やすい**
- → Claude側は「白紙提案より、複数案+理由+推奨」「前提条件を明示してから問う」のが刺さる
- → 「自由に何でも提案して」より「この枠の中で最善を出して」の方が機能する

### ただし制約過剰もNG

- **制約が強すぎるとしんどくなる**
- 適度な制約(Goldilocks原則)がベストパフォーマンス領域
- バランス感覚が大事:
  - 制約ゼロ → 動けない
  - 制約適度 → 最大パフォーマンス
  - 制約過剰 → 苦しい・潰れる
- → Claude側は枠を提示しつつも、Kaina側で調整可能な余白を残すのが理想

## 価値観・信念

- **ファクト志向**: 実証性のないものには乗らない
  - 神話・物語ベースの教義は「そんなんウソじゃん!」判定
- **長期営みへの敬意**: 何百年・千年単位で続いてきた事実そのものに価値を見出す
  - 信仰そのものを信じるかは別軸、"継続している"事実への尊敬
- **短期の派手さより長期成立の仕組みを重視**
  - 「メンテコスト・運用継続性重視」と地続き
- → 二層構造: ファクトで切る目線 × 長期営みへの敬意

## 動機軸: 概念で遊ぶ・記号的観光

- **「そういう場所として定義されている」「そういう体験をすること自体」に価値を置くタイプ**
- 具体例:
  - N40°E140°の経緯度交会点に行く(大潟村)
  - 標高0m基準で作られた人工の山「大潟富士」(高さ4m)に「登山」する
  - 料金所通らず通しで走ってETC料金がいくらになるか観察する(GW東北旅Day5、深夜割引込み10,290円)
- 派手な景観や実用ベネフィットより、**概念・記号・定義された意味付けで遊ぶ**動機
- これは Factorio の工場設計やArsMagiaの仕組み作りとも地続き → **「定義されたルール内で遊ぶ・観察する」系**の動機が一貫してる
- → Claude側はタスク提案時に「こういう枠組みで遊べる」「こういう記号的価値がある」を提示すると刺さる
- 関連: [[../details/drive_context]]

## 動機・駆動軸

- **他者駆動・目的駆動型**(自己駆動は苦手)
  - 「自分のために頑張る」のが苦手
  - **何か理由があった方が頑張れる**タイプ
  - 「誰かのため」「何かのため」が動機源として機能する
- これは他の特性とも統一的に説明できる:
  - 「制約があった方が動ける」(自由度高すぎ苦手)
  - 「対話形式なら思考が出せる」(相手がいれば書ける)
  - 「他者のため」なら奮闘できる(TRPGの推しHO選びにも反映)
- → Claude側はタスク提案時に「誰のためか」「何のためか」を明示すると刺さる
- → マルキン管理職としての立ち位置とも整合(社員のため・会社のためが動機)
