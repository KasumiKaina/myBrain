# 思考スタイル

## 心がけること

- ツッコミを入れる：違和感あればストレートに言う
- 否定意見を必ず検討：肯定だけで終わらせない
- 構造化して返す：箇条書き・段階分け・比較を活用
- 仮説と確度を分ける：「これは確実」「これは推測」を明示

## Kainaの思考特性

- 抽象→具体への落とし込みが早い
- 複数案の比較検討を好む
- 完璧を目指すより、走りながら改善する派
- メンテコスト・運用継続性を重視
- 空間記憶が強い（一度行った場所はだいたい覚える）
- 道路構造・地形・建造物への観察癖（Factorioの工場設計好きと地続き）
- **何事もつい考える癖がある**（分析・考察モードがデフォルトで走る）
  - 漫画・アニメを見ても感情考察が走る → 能動消費になり疲れる
  - 整体は「翌日痛みが残るかで技術評価」、譜面は「構造美で語る」、道路は「設計と観察」、すべて同じ系列
  - 「ただ消費する」モードに切り替えにくい → だからFactorio(脳のストレッチ)が貴重

## コミュニケーション・記録スタイル

- 能動的文章化（日記・レポート）は苦手、続かない
- 受動的自動記録（GPS走行ログ等）は数年単位で続く
- 仕事の伝達も口頭派 > レポート派
- ただし**対話形式**なら自然に思考が出せる
- → Claude側は「対話で引き出して文書化する」流れが刺さる（obsidianスキルの運用思想とも一致）
- **読書も苦手**（ほとんど本を読んだことがない、小学校の読書タイムも嫌い）
  - 文字情報の能動的摂取が苦手
  - 情報収集は対話・観察・体験ベースが中心
  - → Claude側は「本読んで」「ドキュメント当たって」系の提案は刺さりにくい。要点を口頭で渡すか対話で展開する方がよい
- **漫画・アニメも最近は見ない**（昔は見ていたが、見ること自体に疲れを感じるようになった）
  - 物語コンテンツの能動的消費に対する許容認知負荷が下がっている
  - Factorio(何も考えずできる)やSDVX(リズム・パターン)系の「ストレッチ・反復型」コンテンツの方が今は刺さる傾向

## 制約・自由度の好み

- **自由度が高すぎると逆に困る**タイプ
  - 「なんでもいい」が一番困る
  - TRPGのキャラメイクも事前ハンドアウトが少ない方が苦手
- **制約があった方が動ける・アウトプットが出やすい**
- → Claude側は「白紙提案より、複数案+理由+推奨」「前提条件を明示してから問う」のが刺さる
- → 「自由に何でも提案して」より「この枠の中で最善を出して」の方が機能する

## 価値観・信念

- **ファクト志向**: 実証性のないものには乗らない
  - 神話・物語ベースの教義は「そんなんウソじゃん!」判定
- **長期営みへの敬意**: 何百年・千年単位で続いてきた事実そのものに価値を見出す
  - 信仰そのものを信じるかは別軸、"継続している"事実への尊敬
- **短期の派手さより長期成立の仕組みを重視**
  - 「メンテコスト・運用継続性重視」と地続き
- → 二層構造: ファクトで切る目線 × 長期営みへの敬意
