# ArsMagia出先メンテ環境の検討

> created: 2026-04-30
> tags: #判断 #開発環境 #ArsMagia #Codespaces

## 話題

出先でArsMagiaのメンテをやりたい。iPad持ち出してみたけどいまいち上手く取り回せず、周りからはノートPC買えって言われたりもする。何が最適か相談しながら整理した。

## 結論・方針

**A案: GitHub Codespaces + VS Code Web + Claude Code拡張**で試す方向に決定。

普段使ってるVS Code内のClaude Code拡張のワークフローを、Codespaces上のVS Code Webにそのまま移すイメージ。iPad Air M2のSafariで開けば操作感ほぼ同じはず。

検証順序はこれで進める:
1. PCのブラウザでCodespaces立ち上げてClaude Code拡張動作確認
2. PCでnpm run devしてプレビュー確認
3. iPad + 自宅Wi-Fiで同じCodespaceに繋いで操作感確認
4. iPad + テザリングで実環境テスト

各段階で詰まったら戻れるから、一気にやらない。

## 蹴った選択肢

- **ノートPC購入(Surface系/MacBook Air等)**: 月数回の出先利用のために10万円超え + 持ち運びコスト出すのは過剰投資。iPadで済むなら済ませたい。買うのはCodespacesで本当にダメだった時の最終手段。
- **iPad + Working Copy + a-Shell**: ブラウザでのHMRも怪しいし、Node環境がない以上ローカル動作確認できない。根本解決にならない。
- **自宅PC + Tailscale + RDP/Moonlight**: 母艦が落ちてたら詰む、テザリングでの帯域消費が読めない、画面転送方式で重い。既存PC活かせる利点はあるけど不安定要素多い。
- **Claude.ai Web版にコピペ運用**: ファイル操作Claude任せにできない、原始的すぎて効率落ちる。

## 判断軸

- **既存ワークフロー(VS Code + Claude Code拡張)をそのまま移せるか**: ここが最重要。新しい操作感覚えるコストはかけたくない。
- **コスト最小**: 月¥0運用が見込めるなら一番強い。Codespacesは無料枠120 core-hours/月で軽め用途ならタダ。
- **テザリング親和性**: 編集はテキスト通信だけで済むCodespaces方式はRDP系より圧倒的に軽い。
- **メンテコスト・運用継続性**: 端末買い増しは充電・持ち運び・OSアプデ等の運用負担が増える。Codespacesなら端末問わず同じ環境出せる。

## モヤモヤ・揺れ

- Claude Code拡張がVS Code Web/Codespacesで**完全に動作するか**は実測してない。公式サポートはされてるはずだけど、認証フローとか初回ハマりポイントがあるかも。
- テザリングでの**実使用感**は未知数。編集軽い理屈は分かるけど、Claude Code拡張のレスポンス含めると帯域どれくらい食うか読めない。実機テストで判断。
- node_modulesのストレージ消費が地味に気になる。15GB枠内には収まる見込みだけど、複数Codespace作って放置すると食いそう。**使わないCodespaceは削除する運用**を最初に固めとくべきかも。
- 11インチで作業狭い問題は残るけど、これはCodespacesどうこうじゃなくてiPadの物理制約。出先でガッツリ作業するならどうせ無理だから、**軽め修正に割り切る**でOK。

## 気づき・ツッコミ

- **「コーディング自体はClaude主体だからdev立ち上げられればいい」**って気づきが今回の最大のターニングポイント。要件が「フル開発環境」から「Claude+devサーバ+プレビュー」に縮小して、選択肢が一気にシンプルになった。最初の相談時点ではここに気づいてなかった。
- 周りに「ノートPC買え」って言われてホイホイ買わなくて良かった。先にハード買うと**サンクコスト発生して後戻りしづらい**。クラウドサービスは試してダメなら捨てられる、これが正義。
- 「iPadで詰まった具体ポイント」を聞かれた時、**「ローカル動作確認のやり方知らなかっただけかも」**で済んだのが面白い。問題の本質が技術じゃなくて知識だったパターン。
- Codespacesの仕組みを「ブラウザがフロントの仮想マシン」って捉えたら一発で腑に落ちた。RDPと違ってテキストとイベントだけ流すから軽い、っていう構造理解が今後の判断にも効きそう。
- 月¥0で済むなら、出先メンテ用としてだけじゃなく**普段から保険的に契約しといてもいい**かも(ストレージ代だけ月数十円)。母艦壊れた時のバックアップ環境として機能する。
